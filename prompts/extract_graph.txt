
-Goal-
Given an aviation accident report, project proposal, or evaluation plan, identify the most significant entities and causal/structural relationships. Use the "Human-Machine-Environment-Management" (人-机-环-管) framework. Focus on factors that are critical to understanding the document's main purpose or the accident's root causes.

-Entity Types-
- person (人): Pilots, crew members, air traffic controllers, maintenance personnel, engineers, authors, etc.
- equipment (机/设备): Aircraft, engines, avionics, software systems, tools, etc.
- environment (环/环境): Weather conditions, airport conditions, terrain, project constraints, etc.
- organization (组织): Airlines, regulatory bodies, laboratories, companies, departments, etc.
- procedure (程序): Standard Operating Procedures (SOPs), manuals, checklists, project plans, etc.
- event (事件): Specific occurrences (e.g., engine flameout, project kickoff, system deployment).
- weather (天气): Specific meteorological factors.
- violation (违规): Non-compliance with regulations, procedures, or requirements.

-Relationship Types-
- leads to / causes (导致): One event or factor resulting in another.
- limited by (受限于): An entity's performance or state is constrained by another factor.
- violates (违反): A person or action not following a procedure or regulation.
- not executed (未执行): A required procedure or action that was omitted.
- belongs to (属于): An entity is part of an organization or category.
- manages / authors (管理/编写): An organization or person responsible for a document or system.

-Steps-
1. Identify **core** entities. Only extract entities that directly play a causal or structural role in the accident.
- Strictly use the types: person, equipment, environment, organization, procedure, event, weather, violation.
- **Precision First:** Do not extract long descriptive phrases, procedural steps, or generic concepts as entities. If an entity name is longer than 5 words, it is likely too descriptive; simplify it to a noun phrase.
- entity_name: **Normalize.** Use canonical names (e.g., "1号发动机" instead of "右侧发动机 （1号引擎）").
- entity_description: **Concise (MAX 20 words).**
- **Strict Negative Constraints (DO NOT EXTRACT):**
    - Descriptive clauses, full sentences, or abstract concepts.
    - Entities that do not belong to the allowed types.
    - Dates, times, boilerplate.
Format: ("entity"<|><entity_name><|><entity_type><|><entity_description>)
 
2. Identify significant causal or structural relationships.
- **Evidence-Based:** Only extract a relationship if there is clear, explicit text in the source indicating a causal ("caused by", "led to") or structural ("is part of") link. If the relationship is implied, do not extract it.
- relationship_strength: 1-10 (10 being most critical/explicit).
Format: ("relationship"<|><source_entity><|><target_entity><|><relationship_description><|><relationship_strength>)
 
3. **All outputs (names, descriptions) MUST be in the same language as the source text.** Only use the English type labels (person, organization, etc.). Use **##** as the list delimiter.
 
4. When finished, output <|COMPLETE|>
